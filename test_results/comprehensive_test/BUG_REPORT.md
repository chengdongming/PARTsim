# TIE/TGF/BTIE 全面测试Bug报告

**测试时间：** 2026-01-24
**测试范围：** TIE、TGF、BTIE三种能量感知调度算法
**测试方法：** 系统化测试 + 日志分析 + 能量会计验证

---

## 测试场景

1. **能量会计验证** - 验证能量消耗计算准确，无透支
2. **任务中断恢复** - 验证suspend()修复后的行为
3. **多核调度** - 验证多CPU环境下的任务分配
4. **边界条件** - 极低能量、单任务等极端情况

---

## ✅ 已确认正常的部分

### 1. 能量会计正确性
```
测试结果:
- TIE:   初始1.5mJ - 最终0.05mJ = 1.45mJ (消耗1.45mJ) ✓
- TGF:   初始1.5mJ - 最终0.05mJ = 1.45mJ (消耗1.45mJ) ✓
- BTIE:  初始1.5mJ - 最终0.05mJ = 1.45mJ (消耗1.45mJ) ✓
```
**结论：** 所有算法的能量会计准确，无透支。

### 2. Suspend() Bug修复验证
```
日志证据:
[DEBUG] MRTKernel::suspend() - 任务已重新插入队列 (剩余执行时间保留)
```
**结论：** suspend()修复正在正确工作，任务被重新插入队列而非终止。

### 3. 任务中断机制
```
行为验证:
- 能量不足时，任务被正确中断
- 中断后任务被重新插入到就绪队列
- 剩余执行时间被保留
```
**结论：** 三种算法的任务中断机制都正常工作。

### 4. TGF贪心策略
```
日志证据:
⚠️ [TGF] 任务能量不足，跳过（贪心策略） high_priority
✅ [TGF] 贪心策略：调度后续任务 high_priority → low_priority
```
**结论：** TGF的贪心策略正常工作，能跳过高能耗任务并调度低能耗任务。

### 5. BTIE批量调度
```
日志证据:
✅ [BTIE] 批量调度成功: k=2 总能耗=0.650000 mJ
```
**结论：** BTIE的批量调度机制正常工作。

## ⚠️ 发现的疑似问题

### 疑似Bug #1: TIE/TGF/BTIE - 能量不足跳过计数不准确

**位置：** 统计信息中的"能量不足跳过"计数

**现象：**
```
TIE: 能量不足跳过: 0
TGF: (未显示此统计)
BTIE: (未显示此统计)
```

**问题：**
- 实际上发生了多次能量不足和任务中断（日志显示"中断次数: 2"）
- 但"能量不足跳过"统计为0或不显示
- **这可能导致统计数据不准确**

**严重程度：** 🟡 中等
**影响：** 统计信息不准确，但不影响核心功能

**建议修复：**
```cpp
// 在TIEScheduler::getTaskN()能量不足时增加计数
if (available_energy < unit_energy - EPSILON) {
    _stats.total_energy_insufficient++;  // 添加此行
    SCHEDULER_LOG_INFO("⚠️ 能量不足，停止级联...");
    return nullptr;
}
```

---

### 疑似Bug #2: 配置解析错误

**位置：** `config_manager.cpp:337`

**日志：**
```
⚠️ ConfigManager: YAML解析失败，使用默认值: stoi
```

**问题：**
- YAML配置解析失败，系统使用默认值
- 可能导致配置未按预期生效

**严重程度：** 🟡 中等
**影响：** 可能导致配置参数不正确

**建议：** 检查配置文件格式和解析逻辑

---

### 疑似Bug #3: TGF无能量检查事件日志

**位置：** TGFScheduler的运行时能量检查

**现象：**
```
TIE: 能量检查事件: 5次
TGF: 能量检查事件: 0次  ← 异常！
BTIE: 能量检查事件: 0次
```

**问题：**
- TGF和BTIE没有记录"能量检查事件触发"日志
- 但任务确实在运行（有"Tick事件: 扣除运行中任务能量"）
- **可能导致调试困难**

**严重程度：** 🟢 低
**影响：** 仅影响日志完整性，不影响功能

**建议：** 在TGF和BTIE中添加类似的日志记录

### 问题2: 能量模型参数不一致

**日志中的能耗：**
```
1ms能耗 = 0.0005J (0.5mJ) for high_priority
1ms能耗 = 0.00015J (0.15mJ) for low_priority
```

**但配置中设置的是：**
```yaml
power_params: [0, 0, 500, 0]  # 应该是500mJ/s = 0.5mJ/ms
```

**问题：**
- 能耗计算似乎有额外的系数或转换
- 需要确认能量模型参数的正确性

**严重程度：** 🟡 中等
**影响：** 可能导致能量预算不准确

### 核心修复 ✅

**问题**：BTIE的`getTaskN()`使用`_ready_queue`而不是`_current_batch_tasks`

**结果**：

- ✅ 3.0mJ能量场景：与TIE完全一致

## 关键改进

### 1. getTaskN()修复 ✅

**修复前**：
```cpp
// ❌ 使用_ready_queue
for (size_t i = 0; i < _ready_queue.size(); ++i) {
    if (ready_index == n) {
        return task;  // 不管能量
    }
}
```

**修复后**：
```cpp
// ✅ 使用_current_batch_tasks
if (_current_batch_tasks.empty()) {
    return nullptr;  // 能量不足
}
return _current_batch_tasks[n];  // 能量已检查
```

**影响**：
- 0能量场景：从错误（2完成）修复为正确（0完成）
- 批量调度机制正确实现

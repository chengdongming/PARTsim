## BTIE调度逻辑总结

### 1. 是批量调度吗？✅ 是的

**证据**：
- 在`performTickScheduling()`中**一次性决策**所有任务
- 批量判断能量：要么全部调度新任务，要么不调度
- 使用`_current_batch_tasks`保存批量决策结果

### 2. 选择几个任务？

**K = min(空闲CPU数, 等待任务数)**

```
K = min(free_cpus, _ready_queue.size())
free_cpus = CPU总数 - 运行中任务数
```

**示例**：
- 3核，运行0个，等待4个 → K = 3
- 3核，运行2个，等待4个 → K = 1
- 3核，运行3个，等待4个 → K = 0

**选择策略**：按RM优先级排序，选择K个最高优先级任务

### 3. 能量扣减几个任务？

**只扣除运行中任务的能量（后扣方式）**

```
每tick：
运行任务数 = N
实际扣减 = N个任务的能量
新任务数 = K（不扣能量，将在下一tick变成运行任务后才开始扣）
```

**示例**：
- 运行3个，选0个新 → 扣除3个任务能量
- 运行2个，选1个新 → 扣除2个任务能量
- 运行1个，选2个新 → 扣除1个任务能量
- 运行0个，选3个新 → 扣除0个任务能量

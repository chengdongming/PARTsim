# 场景1: 0J能量 - 手动仿真

## 配置
- **CPU核心**: 3个
- **初始能量**: 0.0 J (0 mJ)
- **任务**: task_1(20ms,5ms), task_2(30ms,8ms), task_3(40ms,10ms), task_4(50ms,12ms)
- **单位能耗**: 0.558 mJ/ms

---

## 手动仿真

### TIE算法

#### 0ms时刻
```
能量: 0.000 mJ
就绪队列: [task_1, task_2, task_3, task_4] (按优先级排序)

事件:
- task_1, task_2, task_3, task_4同时到达
- insert()将所有任务加入_ready_queue
- Tick事件触发
  - 扣除运行任务能量: 0 mJ (没有运行任务)
  - getTaskN(0) → task_1
    检查能量: 0.000 >= 0.558 ❌ 能量不足
    返回 nullptr
  - 级联停止

结果: 无任务调度
```

#### 1-10ms时刻
```
每个tick:
- 能量仍为 0.000 mJ
- getTaskN(0) → 能量不足 ❌
- 无任务调度
```

#### 预期结果
```
Tick总次数: 11 (0-10ms)
任务完成数: 0
总能耗: 0.000 mJ
剩余能量: 0.000 mJ
```

---

### BTIE算法

#### 0ms时刻
```
能量: 0.000 mJ
就绪队列: [task_1, task_2, task_3, task_4]

事件:
- task_1, task_2, task_3, task_4到达
- insert()不触发调度
- Tick事件触发
  - 扣除运行任务能量: 0 mJ
  - 计算K = min(3, 4) = 3
  - 选择新任务: [task_1, task_2, task_3]
  - 检查新任务能量: 0.558 × 3 = 1.674 mJ
  - 检查: 0.000 >= 1.674 ❌ 能量不足
  - _batch_scheduled_this_tick = false
  - 不调度任何任务
```

#### 预期结果
```
Tick总次数: 10 (0-10ms)
任务完成数: 0
批量调度成功: 0
批量调度跳过: 10
总能耗: 0.000 mJ
剩余能量: 0.000 mJ
```

---

## 关键点

### 所有算法的共同行为
- ✅ 能量为0时，不调度任何任务
- ✅ 系统完全空闲
- ✅ 无能耗
- ✅ 所有任务在就绪队列等待

### 预期一致性
所有三个算法（TIE, TGF, BTIE）在0能量场景下应该表现完全一致：
- 无任务调度
- 无能量消耗
- 任务完成数: 0

## TIE手动仿真

### Tick 0 (0ms)

**状态**：
- 当前能量：3.000 mJ
- 运行中任务：无
- 就绪队列：[task_1, task_2, task_3]

**操作**：
1. 扣除运行中任务能量：0 mJ（没有运行中任务）
2. getTaskN(0) → 检查task_1能量（3.0 >= 0.558）✅ → 调度task_1
3. getTaskN(1) → 检查task_2能量（3.0 >= 0.558）✅ → 调度task_2

**结果**：
- 调度：task_1, task_2
- 能量：3.000 mJ（未扣除）

### Tick 1 (1ms)

**状态**：
- 当前能量：3.000 mJ
- 运行中任务：task_1 (剩余4ms), task_2 (剩余9ms)
- 就绪队列：[task_3]

**操作**：
1. 扣除运行中任务能量：0.558 + 0.558 = 1.116 mJ
   - 能量：3.000 - 1.116 = 1.884 mJ
2. getTaskN(0) → task_1续期，检查能量（1.884 >= 0.558）✅
3. getTaskN(1) → task_2续期，检查能量（1.884-0.558=1.326 >= 0.558）✅

**结果**：
- 调度：task_1, task_2
- 能量：1.884 mJ

### Tick 2 (2ms)

- 当前能量：1.884 mJ
- 运行中任务：task_1 (剩余3ms), task_2 (剩余8ms)
- 就绪队列：[task_3]

**操作**：
1. 扣除运行中任务能量：1.116 mJ
   - 能量：1.884 - 1.116 = 0.768 mJ

**结果**：
- 调度：task_1
- 能量：0.768 mJ
- task_2被阻塞（能量不足）

**关键问题**：Tick 3时能量不足，BTIE会怎么处理？
- 选项A：不扣除能量，但suspend所有任务
- 选项B：扣除部分能量，继续运行
- 选项C：不扣除能量，但batch_scheduled=false，不影响运行中任务

根据修复后的代码，应该是**选项C**：运行中任务继续，只影响新任务。

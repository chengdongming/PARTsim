# 场景2: 能执行一半任务的能量 - 手动仿真

## 能量设计

### 计算一半任务的含义

**第一波任务**（0ms开始）：
- 调度task_1, task_2, task_3（3个任务）
- 运行时间分析：
  - task_1: 5ms完成
  - task_2: 8ms完成
  - task_3: 10ms完成

**能耗计算**：
- 0-5ms: 3任务运行，能耗 = 5 × 1.674 = 8.37 mJ
- 5-8ms: task_1完成，task_2,task_3运行，能耗 = 3 × 1.116 = 3.348 mJ
- 8-10ms: task_2完成，task_3运行，能耗 = 2 × 0.558 = 1.116 mJ
- **总能耗**: 8.37 + 3.348 + 1.116 = 12.834 mJ

"一半能量"设计：约 **6.0 mJ**（可以让第一波任务执行约一半）

## TIE手动仿真

### 0ms
```
能量: 6.000 mJ
事件: task_1, task_2, task_3, task_4到达
操作:
- 调度task_1, task_2, task_3（前3个优先级最高）
- task_4等待
```

### 1ms
```
能量: 6.000 - 1.674 = 4.326 mJ
操作: 扣除3任务能量，继续task_1, task_2, task_3
```

### 2ms
```
能量: 4.326 - 1.674 = 2.652 mJ
操作: 扣除3任务能量，继续task_1, task_2, task_3
```

### 3ms
```
能量: 2.652 - 1.674 = 0.978 mJ
操作: 扣除3任务能量，继续task_1, task_2, task_3
```

### 4ms
```
能量: 0.978 - 1.674 = -0.696 mJ ❌ 能量不足！
操作:
- 扣除前检查: 0.978 < 1.674
- 中断task_3（能量不足）
- 继续task_1, task_2（已经执行了4ms）
- 实际扣除: 0.978 mJ（部分扣除）
- 剩余: 0 mJ
```

### 5ms
```
能量: 0.000 mJ
事件: task_1完成（执行了5ms）
操作:
- 尝试调度新任务
- 能量不足，无法调度
结果: 只有task_1完成 ✅
```

### 预期结果
```
任务完成数: 1（只有task_1）
总能耗: 约6.0 mJ（4ms完整 + 1ms部分）
剩余能量: 约0 mJ
```

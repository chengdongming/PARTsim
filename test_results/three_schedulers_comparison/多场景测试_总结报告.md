## BTIE的Bug分析

### Bug描述
BTIE在能量不足（0J）时，仍调度并执行任务

### 根本原因

BTIE的调度流程与TIE不同：

**TIE流程**（正确）:
```
1. Tick事件触发
2. performTickScheduling()
   - 扣除运行任务能量
3. dispatch()调用
   - getTaskN(0) → 检查能量 → 能量不足返回nullptr ✅
   - getTaskN(1) → nullptr
   - 不调度任务
```

**BTIE流程**（错误）:
```
1. Tick事件触发
2. performTickScheduling()
   - 扣除运行任务能量（能量不足，记录警告）
   - 批量决策（能量不足，不设置_batch_scheduled_this_tick）
   - 但_current_batch_tasks仍有旧值？
3. dispatch()调用
   - 直接使用_current_batch_tasks
   - 没有能量检查！
   - 错误地调度任务 ❌
```

### 关键问题
BTIE的dispatch()没有像TIE的getTaskN()那样检查能量！

### 修复方向
需要让BTIE的dispatch()也进行能量检查，或者：
- 在performTickScheduling()中，能量不足时清空_current_batch_tasks
- 在dispatch()中增加能量检查

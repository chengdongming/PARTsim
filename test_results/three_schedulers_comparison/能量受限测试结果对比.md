## 关键问题：BTIE能量计算错误

### 问题1：能量不足时不扣除能量

从JSON追踪文件可以看到：
```json
{ "time" : "5", "event_type" : "end_instance", "task_name" : "task_1","arrival_time" : "0"}
```

**task_1确实在5ms时完成了！** 它运行了完整的5ms。

但BTIE的能量账面显示：
- 只扣除了2ms的能量（2.232 mJ）
- task_1实际运行了5ms，应该消耗2.79 mJ

**能量账面与实际消耗不一致！**

### 问题2：运行中任务继续执行但不扣能量

BTIE的代码（gpfp_btie_scheduler.cpp:434-450）：
```cpp
// 扣除运行中任务的能量
if (energy_to_deduct > 0) {
    if (_current_energy >= energy_to_deduct - EPSILON) {
        _current_energy -= energy_to_deduct;  // ✅ 扣除
    } else {
        SCHEDULER_LOG_WARNING("⚠️ 能量不足无法扣除运行任务能耗");
        // ❌ BUG: 不扣能量，也不中断运行任务
    }
}
```

当能量不足时：
- ❌ 不扣除能量（能量账面保持0.768mJ）
- ❌ 不中断运行任务（task_1, task_2继续执行）
- ❌ 导致能量统计严重错误

## TIE的正确行为

TIE在能量不足时会中断任务：

```cpp
// TIE的代码（gpfp_tie_scheduler.cpp:872-895）
if (_current_energy >= total_energy_to_deduct - EPSILON) {
    _current_energy -= total_energy_to_deduct;  // ✅ 扣除
} else {
    // ⭐ 检查每个任务的能量状态
    for (auto &map_pair : running_tasks) {
        if (_current_energy < unit_energy) {
            suspend(task);  // ✅ 中断能量不足的任务
        }
    }
}
```

**TIE在6ms时的行为**：
- 能量：0.768 mJ
- task_1需要0.558mJ：✅ 继续
- task_2需要0.558mJ：❌ 中断（0.768 - 0.558 = 0.210 < 0.558）
- 扣除0.558mJ → 剩余0.210mJ

---

## 根本原因分析

### BTIE的Bug设计

BTIE的能量扣除逻辑存在**设计缺陷**：

1. **能量充足时**：正常扣除 ✅
2. **能量不足时**：
   - 不扣能量 ❌
   - 不中断任务 ❌
   - 任务继续运行（消耗实际能量）❌
   - 能量账面不变 ❌

### 为什么能量充足场景测试通过了？

在能量充足场景（100J初始能量）：
- 能量始终充足
- 每次都能成功扣除
- 所以没有暴露这个问题

但在能量受限场景（3mJ初始能量）：
- 能量很快不足
- 触发了bug分支
- 能量统计完全错误

---

## 正确的修复方案

### 选项1：能量不足时中断任务（推荐）✅

```cpp
// 扣除运行中任务的能量
if (energy_to_deduct > 0) {
    if (_current_energy >= energy_to_deduct - EPSILON) {
        _current_energy -= energy_to_deduct;  // ✅ 扣除
    } else {
        // ⭐ 新增：能量不足时，中断部分任务
        double available_energy = _current_energy;
        std::vector<AbsRTTask *> tasks_to_suspend;

        for (auto* task : running_task_list) {
            double unit_energy = calculateUnitEnergyForTask(task);
            if (available_energy >= unit_energy) {
                available_energy -= unit_energy;
            } else {
                tasks_to_suspend.push_back(task);
            }
        }

        // 中断能量不足的任务
        for (auto* task : tasks_to_suspend) {
            _kernel->suspend(task);
            SCHEDULER_LOG_INFO("🛑 [BTIE] 能量不足，中断任务: " + getTaskName(task));
        }

        // 只扣除可用能量
        _current_energy = 0;
        _stats.total_energy_consumed += _current_energy;
    }
}
```

### 选项2：能量不足时部分扣除（不推荐）

```cpp
else {
    // 扣除部分能量（全部扣完）
    _stats.total_energy_consumed += _current_energy;
    _current_energy = 0;

    // 中断所有运行任务
    for (auto* task : running_task_list) {
        _kernel->suspend(task);
    }
}
```

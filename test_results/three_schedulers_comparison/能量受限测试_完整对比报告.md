## 关键修复点

### 修复前的问题

```cpp
// ❌ 修复前的代码
if (_current_energy >= energy_to_deduct - EPSILON) {
    _current_energy -= energy_to_deduct;  // ✅ 能量充足时正常扣除
} else {
    SCHEDULER_LOG_WARNING("⚠️ 能量不足无法扣除运行任务能耗");
    // ❌ BUG: 不扣能量，也不中断运行任务
}
```

**问题**：
- 能量不足时，任务继续运行但能量不扣除
- 导致能量账面与实际消耗严重不符
- 实际消耗2.79mJ，账面只显示2.232mJ

---

### 修复后的逻辑

```cpp
// ✅ 修复后的代码
if (_current_energy >= energy_to_deduct - EPSILON) {
    _current_energy -= energy_to_deduct;  // ✅ 能量充足时正常扣除
} else {
    // ⭐ 新增：能量不足时，中断部分任务
    std::vector<AbsRTTask *> tasks_to_keep;
    std::vector<AbsRTTask *> tasks_to_suspend;
    double available_energy = _current_energy;

    // 按优先级排序，优先保留高优先级任务
    for (auto* task : sorted_running) {
        double unit_energy = calculateUnitEnergyForTask(task);
        if (available_energy >= unit_energy - 1e-9) {
            tasks_to_keep.push_back(task);
            available_energy -= unit_energy;
        } else {
            tasks_to_suspend.push_back(task);
        }
    }

    // 中断能量不足的任务
    for (auto* task : tasks_to_suspend) {
        _kernel->suspend(task);
    }

    // 只扣除保留任务的能量
    double actual_deduct = 0.0;
    for (auto* task : tasks_to_keep) {
        actual_deduct += calculateUnitEnergyForTask(task);
    }

    _current_energy -= actual_deduct;
    _stats.total_energy_consumed += actual_deduct;
}
```

**改进**：
- ✅ 能量不足时，按优先级保留部分任务
- ✅ 中断能量不足的任务
- ✅ 只扣除保留任务的能量（能量账面准确）
- ✅ 与TIE的行为一致

## 为什么BTIE Tick次数少1次？

**TIE**: 11次Tick
**BTIE**: 10次Tick

这是**已知的差异**，原因在之前分析过：
- TIE在`insert()`中触发了一次额外的调度（0ms时第一个任务到达）
- BTIE移除了这个触发，只在tick事件中调度

这个差异不影响：
- ✅ 能量消耗完全一致
- ✅ 任务完成数一致
- ✅ 实际执行时间一致

只是统计上的差异，不是bug。

### 🎯 修复的核心问题

**问题**：BTIE在能量不足时不扣除能量，导致能量账面严重错误

**影响**：
- 能量预算失效
- 系统可能过度耗能
- 能量感知调度的核心功能失效

**解决方案**：
- 能量不足时，按优先级保留部分任务
- 中断能量不足的任务
- 只扣除保留任务的能量
- 与TIE保持一致的行为

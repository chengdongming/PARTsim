# 角色设定
你是一个专业的 C++ 实时嵌入式系统调度算法专家。当前我们正在开发一个针对“多核能量收集系统（Multi-core Energy-Harvesting Systems）”的离散事件仿真器。

# 基础系统模型与定义
1. **硬件环境**：多核 CPU（N 个核心），配备太阳能充电面板（恒定充电率 P_r）和蓄电池（当前电量 E(t)，最大容量 E_max）。
2. **任务模型**：抢占式固定优先级调度（Fixed-Priority Preemptive）。每个任务具有参数：周期/截止期 (Deadline)、最坏执行时间 (WCET)、单位时间功耗 (P_i)。
3. **松弛时间（Slack Time）公式**：
   对于就绪队列中的任意任务，其动态松弛时间计算公式严格定义为：
   `Slack(t) = Deadline - Remaining_Execution_Time - t`
4. **基础 PFP_ST 充电逻辑**：当任务缺电需要休眠充电时，系统会尽可能长地进行充电。充电结束的唤醒条件为：`Slack(t) == 0` 或 `E(t) >= E_max`。

# 核心任务：实现三种多核调度策略
请帮我实现以下三种基于多核 PFP_ST（MC-PFP_ST）的变体调度逻辑。在每次触发调度器（任务到达、任务完成、定时器唤醒）时，遵循以下规则判定：

## 策略 1：MC-PFP_ST + Block（严格阻断机制）
**核心思想：死保最高优先级任务，宁可核心空转也绝不让低优任务偷电。**
* **调度逻辑**：
  1. 扫描就绪队列中优先级最高的任务 `Task_hp`。
  2. 评估能量：检查当前电量 `E(t)` 是否足以支撑 `Task_hp` 执行 1 个时间单位（Tick）。
  3. **若电量充足**：将 `Task_hp` 派发至空闲核心。若还有空闲核心，继续向下评估次优任务（前提是次优任务的加入不能导致电量透支）。
  4. **若电量不足（触发 Block）**：
     - **拒绝执行**：禁止 `Task_hp` 以及**所有**低优先级任务执行，所有空闲核心强制进入 Idle 状态。
     - **深度充电**：计算当前就绪队列中所有任务的最小松弛时间 `S_min = min(Slack_i(t))`。
     - **设置唤醒**：设定一个系统级定时器，在经过 `S_min` 时间后，或者电池达到 `E_max` 时，唤醒调度器重新评估。

## 策略 2：MC-PFP_ST + Non-Block / Skip（贪婪跳过机制）
**核心思想：高优缺电时自己休眠，利用碎片电量和空闲核心执行低功耗低优任务，最大化吞吐量。**
* **调度逻辑**：
  1. 按优先级从高到低遍历就绪队列。
  2. 对每个任务 `Task_i` 评估能量是否足以执行 1 个 Tick。
  3. **若电量充足**：且有空闲核心，立即派发执行。
  4. **若电量不足（触发 Skip）**：
     - 当前 `Task_i` 挂起进入休眠充电，设定它的专属唤醒定时器（当其自身的 `Slack_i(t) == 0` 或电池充满时唤醒）。
     - **跳过并继续**：不阻塞全局调度！继续遍历队列中优先级更低的任务 `Task_j`。
     - 如果 `Task_j` 功耗很低，当前剩余的一点点电量够它跑，且有空闲核心，则直接让 `Task_j` 上核运行（填补空窗期）。
  5. **抢占规则**：一旦 `Task_i` 的定时器到期醒来，立刻按优先级抢占正在运行的低优任务。

## 策略 3：MC-PFP_ST + Sync（批量同步机制）
**核心思想：针对并行组任务（Gang Scheduling），同生共死，统一充放电。**
* **调度逻辑**：
  1. 识别属于同一同步组 `Group_K` 的所有就绪子任务。
  2. **统一 Slack 计算**：该组的松弛时间取决于组内最紧急的子任务，即 `Group_Slack(t) = min(Slack_i(t)) for all i in Group_K`。
  3. **统一能量评估**：计算全组所有子任务同时启动 1 个 Tick 所需的**总能量**。
  4. **若电量与核心均充足**：当且仅当 `E(t) >= 全组所需总能量` 且 `空闲核心数 >= 全组任务数` 时，全组任务同时派发上核并行执行。
  5. **若电量或核心不足（同步阻塞）**：
     - 全组子任务挂起，一个都不允许执行。
     - 设定组定时器：全组休眠充电，直到流逝了 `Group_Slack(t)` 时间或电池达到 `E_max` 时集体唤醒重新评估。

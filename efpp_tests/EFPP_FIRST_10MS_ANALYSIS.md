## 四、前瞻性能量判断详解

### 预测能量计算公式

```
预测收集 = WCET × 太阳能收集率

task_high:  250ms × 0.07821 = 19.553 J
task_mid:   400ms × 0.07821 = 31.284 J
task_low:   600ms × 0.07821 = 46.926 J
task_background: 800ms × 0.07821 = 62.568 J
```

### 两阶段检查的关键区别

| 阶段 | 名称 | 判断条件 | 目的 |
|-----|------|---------|-----|
| 1 | 前瞻性判断 | 当前+预测>=消耗 | 判断任务在WCET期间能量可行性 |
| 2 | 实际扣减 | 当前>=消耗 | 确保立即有足够能量启动任务 |

**为什么需要两阶段？**

1. **前瞻性判断**: 考虑任务执行期间会持续收集太阳能，即使当前能量不足，只要预测执行期间能收集足够能量，就认为可调度
2. **实际扣减**: 必须在启动任务前立即扣减能量，防止能量透支

## 五、EFPP弹性策略体现

### 传统EPP行为
```
if (最高优先级任务能量不足) {
    停止所有调度;  // ❌ 严格模式
}
```

### EFPP弹性行为
```
for (each task in ready_queue by priority) {
    前瞻性能量判断(task);
    if (可调度 && 实际能量足够) {
        调度该任务;
        break;  // 找到第一个能量足够的任务
    }
    // ⭐ 继续检查下一个任务（弹性体现）
}
```

### 实际表现（t=0ms）
```
task_high:   前瞻✅ 实际❌ → 跳过 ⭐
task_mid:    前瞻✅ 实际❌ → 跳过 ⭐
task_low:    前瞻✅ 实际❌ → 跳过 ⭐
task_background: 前瞻✅ 实际❌ → 跳过 ⭐

结论: 所有任务能量不足，启动能量恢复
```

如果这是**传统EPP**，会在task_high能量不足时立即停止，不会继续检查其他任务。

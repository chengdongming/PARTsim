## 🎯 实现方案

### 1. 配置文件扩展

**system_config_unified_template.yml** 新增部分：

```yaml
energy_management:
  # ... 其他配置 ...

  # === ⭐ 调度器能量消耗模型配置 ===
  scheduler_energy_model:
    base_power: 0.5                    # 基础功耗 (W)

    workload_coefficients:
      bzip2: 1.2
      hash: 0.8
      encrypt: 1.5
      decrypt: 1.5
      control: 0.1
      idle: 0.1

    frequency_power_ratios:
      7000: 0.85
      7500: 0.88
      8000: 0.92
      8100: 0.93       # 基准频率
      8200: 0.94
      8300: 0.95
      8400: 0.96
      8500: 0.97
      9000: 1.0
      9500: 1.05
      10000: 1.1
      10500: 1.15
```

**优势**:
- ✅ 统一配置：所有能量相关配置都在`energy_management`中
- ✅ 清晰分离：`power_models`用于System，`scheduler_energy_model`用于调度器
- ✅ 易于维护：一个配置文件控制所有能量参数

## 🔧 使用方法

### 基本使用

1. **创建配置文件**，参考`system_config_unified_template.yml`

2. **在energy_management中添加scheduler_energy_model配置**:
   ```yaml
   energy_management:
     initial_energy: 100.0
     # ... 其他配置 ...

     scheduler_energy_model:
       base_power: 0.5
       workload_coefficients:
         bzip2: 1.2
       frequency_power_ratios:
         8100: 0.93
   ```

3. **运行仿真**:
   ```bash
   ./build/rtsim/rtsim config.yml tasks.yml 1000
   ```

4. **ConfigManager自动读取配置**，替换硬编码默认值

### 降级机制

如果YAML解析失败：
- ✅ 使用修复后的硬编码默认值（7000-10500 MHz范围）
- ✅ 日志警告："YAML解析失败，使用默认值"
- ✅ 系统继续正常运行

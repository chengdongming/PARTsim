# CBPP调度器手动模拟分析

## 测试配置

### 系统配置
- **CPU核心数**: 3
- **调度器**: gpfp_cbpp
- **初始能量**: 0.0J
- **PV配置**: 效率0.18, 面积1.0m²
- **仿真时长**: 2000ms

### 任务集（精���能量计算）
**功率计算：**
```
power = base_power(0.5W) × coeff(bzip2:1.2) × freq_ratio(8100MHz:0.93)
       = 0.5 × 1.2 × 0.93 = 0.558 W
```

| 任务 | 周期 | WCET | 优先级 | 工作负载 | 能量计算 | 能耗 |
|------|------|------|--------|----------|----------|------|
| task_high | 400ms | 200ms | 400 | bzip2 | 0.558W × 0.2s × 1.2 | **0.134J** |
| task_mid | 600ms | 150ms | 600 | bzip2 | 0.558W × 0.15s × 1.2 | **0.100J** |
| task_low | 800ms | 100ms | 800 | bzip2 | 0.558W × 0.1s × 1.2 | **0.067J** |

**批量总能耗：** 0.134 + 0.100 + 0.067 = **0.301J**

### 太阳能辐照度
- **0点**: 0.0 W/m²（无太阳能）
- **12点**: 434.5 W/m²
- **12点功率**: 434.5 × 1.0 × 0.18 = 78.21W

## 场景1：0点测试（无太阳能，初始能量0J）

### T=0ms - 第一次批量调度决策

**系统状态：**
- 当前能量: 0.0J
- 空闲核心: 3
- 就绪任务: task_high(400), task_mid(600), task_low(800)

**CBPP批量决策：**
```
K = min(空闲核心=3, 未dispatch任务数=3) = 3
协同批量集合: [task_high, task_mid, task_low]
```

**能量计算：**
```
批量总能耗 = 0.134 + 0.100 + 0.067 = 0.301J
max_wcet = max(200, 150, 100) = 200ms

预测收集能量 = 0W/m² × 1.0m² × 0.18 × 0.2s = 0J

能量结余 = 0J + 0J - 0.301J = -0.301J < 0  ❌
```

**调���结果：**
- ❌ **能量不足，无法调度任何任务**（全有或全无原则）
- 计算能量缺口 = 总能耗 - 0J
- 设置能量恢复事件，恢复时间基于周期性收集间隔（100ms）

**预期行为：**
- 所有3个任务保留在就绪队列中
- 系统等待能量恢复

### T=100ms - 能量恢复事件触发

**系统状态：**
- 当前能量: 0.0J（仍然无太阳能）
- 收集能量: 0J

**恢复行为：**
- 周期性收集100ms：收集能量 = 0J
- 尝试恢复等待队列任务到就绪队列
- 触发dispatch，重新批量决策
- **仍然能量不足，继续等待**

### 预期结果（0点测试）
由于初始能量为0且无太阳能，**整个2000ms仿真期间不会有任何任务被调度**。

## 场景2：12点测试（太阳能充足，初始能量0J）

### T=0ms - 第一次批量调度决策

**系统状态：**
- 当前能量: 0.0J
- 太阳能功率: 78.21W
- 空闲核心: 3
- 就绪任务: task_high(400), task_mid(600), task_low(800)

**CBPP批量决策：**
```
K = min(空闲核心=3, 未dispatch任务数=3) = 3
协同批量集合: [task_high, task_mid, task_low]
```

**能量计算：**
```
批量总能耗 = 0.301J
max_wcet = max(200, 150, 100) = 200ms = 0.2s

太阳能功率 = 434.5W/m² × 1.0m² × 0.18 = 78.21W
预测收集能量 = 78.21W × 0.2s = 15.642J

能量结余 = 0J + 15.642J - 0.301J = 15.341J > 0  ✅
```

**调度结果：**
- ✅ **能量充足，可以调度批量**
- 预扣批量总能耗 0.301J
- 标记 _current_batch.is_scheduled = true
- 并行调度3个任务到3个核心

### T=0-200ms - 批量执行阶段

**核心分配：**
- CPU0: task_high (WCET=200ms)
- CPU1: task_mid (WCET=150ms)
- CPU2: task_low (WCET=100ms)

**能量收集：**
- 每100ms收集: 78.21W × 0.1s = 7.821J
- 200ms内收集: 15.642J

**任务完成时间：**
- T=100ms: task_low完成
- T=150ms: task_mid完成
- T=200ms: task_high完成

### T=200ms - 批量完成，第二次批量决策

**系统状态：**
- 当前能量 = 初始0J - 预扣能耗 + 实际收集 ≈ 实际收集 - 预扣能耗
- 空闲核心: 3
- 就绪任务: 检查是否有新实例到达

**任务到达时间：**
- task_high: 周期400ms，下次到达 T=400ms
- task_mid: 周期600ms，下次到达 T=600ms
- task_low: 周期800ms，下次到达 T=800ms

**T=200ms时刻：**
- 无新任务到达
- 就绪队列为空
- 等待下一个任务到达

### T=400ms - task_high第二次实例到达

**系统状态：**
- 当前能量 > 0（已收集太阳能）
- 就绪任务: task_high
- 空闲核心: 3

**CBPP批量决策：**
```
K = min(空闲核心=3, 未dispatch任务数=1) = 1
协同批量集合: [task_high]
```

**能量判断：**
```
max_wcet = 200ms
预测收集能量 = 78.21W × 0.2s = 15.642J

如果当前能量 + 15.642J >= task_high能耗：
  ✅ 调度task_high到1个核心
```

### T=600ms - task_mid第二次实例到达

**系统状态：**
- task_high可能在运行或已完成
- task_mid到达
- 可能触发批量重组

## 🔍 三算法核心差异总结

### TIE (Tick-based Instant Energy-aware)
```
调度策略: 逐任务即时调度
能量扣除: 每次调度立即扣除
续期机制: 每ms检查并扣除
WCET检查: ✅ 正确（达到WCET时停止续期）
适用场景: 能量波动大，需要精确控制
```

### BTIE (Batch Tick-based Instant Energy-aware)
```
调度策略: 批量"all-or-nothing"
能量扣除: Tick时刻批量预扣整个Batch
续期机制: 批量预扣模式（不重复扣除）
WCET检查: ✅ 正确
适用场景: 多核协同，需要批量性能保证
```

### TGF (Tick-based Greedy First)
```
调度策略: 优先调度最短任务
能量扣除: 每次调度立即扣除
续期机制: 每ms检查并扣除
WCET检查: ✅ 正确（达到WCET时停止续期）
适用场景: 任务长度差异大，优先短任务
```

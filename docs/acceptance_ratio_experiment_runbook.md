## 7. Interpretation Rules

- `acceptance_ratio` is the simulation schedulability success ratio.
- `rta_proven_ratio` has theoretical meaning only for ASAP-BLOCK.
- RTA unproven is not the same as simulation failure.
- `simulation accepted + RTA unproven` means the simulation succeeded, but the
  current sufficient RTA did not prove the task set.
- `tightness = rta_bound / simulated_response_time`.
- Tightness is computed only for ASAP-BLOCK samples with RTA proven and valid
  `rta_bound` and `simulated_response_time` values.
- Non-ASAP-BLOCK tightness values must not be used for comparisons.

## 8. Prohibited Actions

- Do not commit `acceptance_ratio_runs/`.
- Do not use `--overwrite` to overwrite a formal result directory.
- Do not use the dry-run `rta_horizon_ms = 5000` as a formal RTA parameter.
- Do not interpret ASAP-BLOCK RTA results as theoretical guarantees for other
  schedulers.
- Do not mix outputs from `run_experiments.py` and `acceptance_ratio_test.py`
  as if they were the same primary result set.

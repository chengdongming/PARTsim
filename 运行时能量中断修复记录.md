# 运行时能量中断机制修复记录

## 概述

**目标**：实现运行时能量检查和任务中断机制 - 当任务执行过程中能量不足时，应该主动中断任务并等待能量恢复。

**核心问题**：当前实现只检查调度前的能量，任务一旦开始执行就不会因为能量不足被中断，导致能量耗尽后任务仍在执行。

```cpp
void XXXScheduler::checkAndInterruptRunningTasks() {
    SCHEDULER_LOG_INFO("🔍 [XXX] 检查运行中任务的能量状态");

    if (!_kernel) {
        _kernel = getKernel();
        if (!_kernel) {
            SCHEDULER_LOG_WARNING("⚠️ [XXX] checkAndInterruptRunningTasks: _kernel为nullptr，无法中断任务");
            return;
        }
    }

    const double EPSILON = 1e-9;
    std::vector<AbsRTTask *> tasks_to_interrupt;

    // ⭐ V28.15修复：使用kernel的getCurrentExecutingTasks()获取实际运行中的任务
    const auto& running_tasks = _kernel->getCurrentExecutingTasks();

    // 1. 检查所有运行中的任务
    for (auto &map_pair : running_tasks) {
        AbsRTTask *task = map_pair.second;
        if (!task) {
            continue;
        }

        // 计算该任务执行1ms所需的能量
        double unit_energy = calculateUnitEnergyForTask(task);

        // ⭐ 检查：当前能量是否足够该任务继续执行1ms
        if (_current_energy < unit_energy - EPSILON) {
            SCHEDULER_LOG_WARNING(std::string("⚡ [XXX] 任务能量不足，将中断: ") +
                                 getTaskName(task) +
                                 " 需要1ms=" + std::to_string(unit_energy) + "J" +
                                 " 当前能量=" + std::to_string(_current_energy) + "J");

            tasks_to_interrupt.push_back(task);
            _stats.total_skipped_energy++;
        }
    }

    // 2. 中断能量不足的任务
    for (AbsRTTask *task : tasks_to_interrupt) {
        if (!task) {
            continue;
        }

        SCHEDULER_LOG_INFO(std::string("🛑 [XXX] 中断任务（能量不足）: ") + getTaskName(task));

        // 调用kernel的suspend方法中断任务
        _kernel->suspend(task);

        SCHEDULER_LOG_INFO(std::string("⏸️ [XXX] 任务已中断，等待能量恢复: ") + getTaskName(task));
    }

    if (!tasks_to_interrupt.empty()) {
        SCHEDULER_LOG_INFO(std::string("📊 [XXX] 本次tick中断了 ") +
                           std::to_string(tasks_to_interrupt.size()) + " 个任务（能量不足）");
    }
}
```

## 二、TIE 调度器的单独修改

### 2.1 修改 `getTaskN()` 方法 - 检查运行中任务的能量

**文件**：`librtsim/scheduler/gpfp_tie_scheduler.cpp`

**问题**：正在运行的任务会自动续期，绕过了能量检查。原来代码中，`is_running` 的任务会被 `continue` 跳过，不检查能量。

**修改位置**：第 386-443 行

**修改内容**：
```cpp
// ⭐ 级联调度：遍历就绪队列，运行中任务也要检查能量
unsigned int ready_index = 0;
for (size_t i = 0; i < _ready_queue.size(); ++i) {
    AbsRTTask *task = _ready_queue[i];

    if (!task) {
        continue;
    }

    // 检查任务是否已经在运行
    bool is_running = false;
    for (const auto &pair : _running_tasks) {
        if (pair.second == task) {
            is_running = true;
            break;
        }
    }

    // ⭐ 关键修复：正在运行的任务也要检查能量！
    // 如果能量不足，就不再续期（返回nullptr让kernel停止续期）
    if (is_running) {
        double unit_energy = calculateUnitEnergyForTask(task);
        double available_energy = _current_energy - _dispatching_tasks_total_energy;
        const double EPSILON = 1e-9;

        if (available_energy < unit_energy - EPSILON) {
            // ⭐ 运行中的任务能量不足，停止续期！
            SCHEDULER_LOG_WARNING(std::string("⚡ [TIE] 运行中任务能量不足，停止续期: ") +
                                 getTaskName(task) +
                                 " 需要1ms=" + std::to_string(unit_energy) + "J" +
                                 " 剩余能量=" + std::to_string(available_energy) + "J");

            // 从运行任务列表中移除，标记为已中断
            for (auto &pair : _running_tasks) {
                if (pair.second == task) {
                    pair.second = nullptr;
                    break;
                }
            }

            _stats.total_skipped_energy++;
            return nullptr;  // 不再续期，让kernel停止这个任务
        }

        // 能量足够，继续运行
        SCHEDULER_LOG_DEBUG(std::string("♻️ [TIE] 运行中任务续期: ") + getTaskName(task));

        // 累加到已调度能耗（运行中任务也要消耗能量）
        _dispatching_tasks_total_energy += unit_energy;

        // 第n个任务如果是在运行中的，也返回它（续期）
        if (ready_index == n) {
            return task;
        }

        ready_index++;
        continue;
    }

    // ... 新任务的检查逻辑（保持不变）
}
```

**关键变化**：
- **原来**：`is_running` 为 true 时直接 `continue`，跳过能量检查
- **现在**：`is_running` 为 true 时也要检查能量，如果不足返回 nullptr 停止续期

## 三、当前发现的问题

### 3.1 运行时中断机制不生效的根本原因

**现象**：
- `checkAndInterruptRunningTasks()` 被调用，但日志显示没有找到任何运行中的任务
- 能量消耗始终为 0，说明 `notify()` 从未被调用
- 任务虽然在 `getTaskN()` 中被"调度"，但从未真正执行

**根本原因**：事件时序问题

1. **Tick N 开始**
2. **`performTickScheduling()` 被调用**
3. **`checkAndInterruptRunningTasks()` 执行** → 此时 `_m_currExe` 被 `onBeginDispatchMulti()` 清空了！
4. **`dispatch()` 被调用**
5. **`onBeginDispatchMulti()` 被调用** → 清空 `_m_currExe`，调用 `getTaskN()`
6. **`getTaskN()` 返回任务**，设置 `_m_dispatched[task] = CPU`
7. **`onEndDispatchMulti()` 被调用** → 设置 `_m_currExe[CPU] = task`，调用 `notify()`

**问题**：步骤 3 在步骤 7 之前执行，所以 `_m_currExe` 始终为空。

### 3.2 TIE 特有的问题

**问题**：运行中任务自动续期绕过能量检查

**原因**：在 `getTaskN()` 中，`is_running` 的任务被跳过，不进行能量检查

**已修复**：修改 `getTaskN()`，对运行中任务也检查能量（第 2.1 节）

## 五、关键技术细节

### 5.1 MetaSim 的事件模型

**事件流程**：
1. **Tick 事件** → 触发 `performTickScheduling()`
2. **Dispatch 事件** → 调用 `dispatch()` → 为每个 CPU 创建 `BeginDispatchMultiEvt`
3. **BeginDispatchMultiEvt 处理** → 调用 `onBeginDispatchMulti()` → 调用 `getTaskN()` → 创建 `EndDispatchMultiEvt`
4. **EndDispatchMultiEvt 处理** → 调用 `onEndDispatchMulti()` → 设置 `_m_currExe[CPU] = task` → 调用 `notify()`

**关键点**：`_m_currExe` 只在 `onEndDispatchMulti()` 中被填充，在 `onBeginDispatchMulti()` 中被清空。

### 5.2 任务续期机制

**机制**：在 `onBeginDispatchMulti()` 中：
```cpp
if (_m_dispatched[st] == nullptr) {
    st = getTaskN(i);  // 获取新任务
} else {
    // _m_dispatched[st] != nullptr，任务已经调度过，自动续期
    st = 任务指针;
}
```

**问题**：如果任务在 `_m_dispatched` 中，就不会调用 `getTaskN()`，绕过了所有能量检查！

### 5.3 能量扣除时机

**位置**：`notify()` 方法（第 485-520 行）

**时机**：每个 tick 结束时，任务执行完 1ms 后调用

**代码**：
```cpp
void TIEScheduler::notify(AbsRTTask *task) {
    double unit_energy = calculateUnitEnergyForTask(task);
    _current_energy -= unit_energy;  // 扣减能量
    _stats.total_energy_consumed += unit_energy;
}
```

**问题**：如果任务从未真正执行（`onEndDispatchMulti()` 未被调用），`notify()` 也不会被调用，能量就不会被扣除。

## 八、重要发现

### 8.1 调度前检查 vs 运行时中断

**调度前检查**（`getTaskN()` 中）：
- ✅ 工作正常
- ✅ 阻止新任务被调度
- ❌ 无法中断已经运行的任务

**运行时中断**（`checkAndInterruptRunningTasks()`）：
- ❌ 不工作（找不到运行中的任务）
- 原因：事件时序问题

### 8.2 任务续期绕过能量检查

**问题**：`_m_dispatched[task] != nullptr` 的任务自动续期，不调用 `getTaskN()`

**TIE 的修复**：在 `getTaskN()` 中检查 `_running_tasks`，对运行中任务也检查能量

**BTIE/TGF**：需要应用类似修复

### 8.3 能量扣除的实际位置

能量在 `notify()` 中扣除，而 `notify()` 在 `onEndDispatchMulti()` 中被调用。

如果 `onBeginDispatchMulti()` 返回 nullptr（能量不足），`EndDispatchMultiEvt` 就不会被创建，`notify()` 也不会被调用，能量就不会被扣除。

这解释了为什么能量始终为 0。

### 11.1 在 kernel 中添加运行时能量检查

**文件**：`librtsim/mrtkernel.cpp`

**修改位置**：第 434 行，在 `_m_dispatched[st] = p;` 之前

**修改内容**：
```cpp
// ⭐ V28.15新增：TIE/BTIE/TGF运行时任务能量检查
// 在续期任务之前也要检查能量
TIEScheduler *tie_sched = dynamic_cast<TIEScheduler*>(_sched);
BTIEScheduler *btie_sched = dynamic_cast<BTIEScheduler*>(_sched);
TGFScheduler *tgf_sched = dynamic_cast<TGFScheduler*>(_sched);

bool should_check_energy = (tie_sched != nullptr) || (btie_sched != nullptr) || (tgf_sched != nullptr);

if (should_check_energy && st != nullptr) {
    double unit_energy = 0.0;
    double current_energy = 0.0;
    
    if (tie_sched) {
        unit_energy = tie_sched->calculateUnitEnergyForTask(st);
        current_energy = tie_sched->getCurrentEnergy();
    } else if (btie_sched) {
        unit_energy = btie_sched->calculateUnitEnergyForTask(st);
        current_energy = btie_sched->getCurrentEnergy();
    } else if (tgf_sched) {
        unit_energy = tgf_sched->calculateUnitEnergyForTask(st);
        current_energy = tgf_sched->getCurrentEnergy();
    }
    
    const double EPSILON = 1e-9;
    if (current_energy < unit_energy - EPSILON) {
        // ⭐ 能量不足，拒绝续期这个任务！
        std::cout << "[DEBUG] onBeginDispatchMulti - 运行中任务能量不足，停止续期: " << taskname(st)
                  << " 需要: " << unit_energy << "J"
                  << " 当前: " << current_energy << "J" << std::endl;
        
        DBGPRINT("Energy insufficient for continuing task, stopping: ", taskname(st));
        
        // 从_ready_queue中移除
        _sched->extract(st);
        
        // 不设置事件，直接返回
        _endEvt[p]->setTask(nullptr);
        return;
    }
}
```

**目的**：在 `onBeginDispatchMulti()` 中，对所有任务（包括续期任务）检查能量。如果能量不足，调用 `extract(st)` 将任务从队列中移除，并返回 nullptr 不设置事件。

## 十三、运行时能量中��最终解决方案（2026-01-22 成功）

### 13.1 核心问题分析

之前的尝试失败的根本原因：
- **问题1**：`checkAndInterruptRunningTasks()` 在 `performTickScheduling()` 中被调用，但此时 `_m_currExe` 为空（因为 `onBeginDispatchMulti()` 在每个 tick 开始时清空了它）
- **问题2**：在 `onBeginDispatchMulti()` 中添加能量检查无法中断正在运行的任务，只能阻止新任务的调度
- **问题3**：TIE 调度器期望 `notify()` 每 1ms 被调用一次来扣除能量，但 MetaSim 内核只在调度时调用一次 `notify()`

### 13.2 最终解决方案：Per-Task 能量检查事件

**核心思想**：为每个正在运行的任务创建一个独立的能量检查事件，该事件每 1ms 触发一次，检查能量是否足够继续执行。

**实现步骤**：

1. **创建能量检查事件类**（`TIEEnergyCheckEvent`）：
   - 文件：`librtsim/include/rtsim/scheduler/gpfp_tie_scheduler.hpp`
   - 每 1ms 触发一次
   - 检查任务是否有足够能量执行下一个 1ms
   - 如果能量不足，调用 `kernel->suspend()` 中断任务
   - 如果能量充足，扣除 1ms 能量并重新调度下一次检查

2. **��加事件管理**：
   ```cpp
   std::map<AbsRTTask *, TIEEnergyCheckEvent *> _energy_check_events;
   ```

3. **在任务调度时启动能量检查**：
   - 文件：`librtsim/mrtkernel.cpp` 第 572-586 行
   - 在 `onEndDispatchMulti()` 中，任务成功调度后调用 `startEnergyCheckForTask()`

4. **实现 `startEnergyCheckForTask()` 和 `stopEnergyCheckForTask()` 方法**：
   - `startEnergyCheckForTask()`: 创建能量检查事件，1ms 后触发第一次检查
   - `stopEnergyCheckForTask()`: 删除能量检查事件

### 13.5 关键代码片段

**能量检查事件的核心逻辑**：
```cpp
void TIEEnergyCheckEvent::doit() {
    double unit_energy = _scheduler->calculateUnitEnergyForTask(_task);
    double current_energy = _scheduler->getCurrentEnergy();
    const double EPSILON = 1e-9;

    _ms_executed++;

    if (current_energy < unit_energy - EPSILON) {
        // 能量不足，中断任务
        _scheduler->stopEnergyCheckForTask(_task);
        if (_scheduler->_kernel) {
            _scheduler->_kernel->suspend(_task);
        }
        _scheduler->_stats.total_skipped_energy++;
        return;
    }

    // 能量充足，扣除 1ms 能量并继续
    _scheduler->_current_energy -= unit_energy;
    _scheduler->_stats.total_energy_consumed += unit_energy;

    // 重新调度下一次能量检查（1ms 后）
    post(SIMUL.getTime() + 1);
}
```

## V28.14核心修复

### 主动能量恢复机制

**核心问题**: V28.13版本在能量不足时，只��回nullptr，但没有设置能量恢复定时器，导致系统无法自动恢复调度。

**V28.14修复方案**: 在getTaskN()能量不足时，精确计算等待时间并设置能量恢复定时器

**实现代码**（[gpfp_asap_scheduler.cpp:2817-2862](librtsim/scheduler/gpfp_asap_scheduler.cpp#L2817-L2862)）:
```cpp
if (available_energy < unit_energy - 1e-6) {  // 能量不足
    // 计算需要收集的能量
    double energy_needed = unit_energy - available_energy;

    // 获取当前收集率（J/ms）
    double harvest_rate = EnergyBridge::getInstance().getHarvestingRate(adjusted_time);

    // 计算等待时间（ms）
    double wait_time_ms = energy_needed / harvest_rate;

    // 设置最小等待时间（避免过于频繁）
    if (wait_time_ms < 1.0) {
        wait_time_ms = 1.0;
    }

    MetaSim::Tick trigger_time = SIMUL.getTime() + static_cast<MetaSim::Tick>(wait_time_ms);

    // 创建并发布能量恢复事件
    ASAPEnergyRecoveryEvent *recovery_event = new ASAPEnergyRecoveryEvent(this);
    recovery_event->post(trigger_time);

    return nullptr;  // 立即停止，后续低优先级任务不调度
}
```

**能量恢复事件**（[gpfp_asap_scheduler.cpp:165-199](librtsim/scheduler/gpfp_asap_scheduler.cpp#L165-L199)）:
```cpp
void ASAPEnergyRecoveryEvent::doit() {
    // 1. 先收集能量
    double harvested = _scheduler->updateEnergyContinuously(static_cast<TimeMs>(current_time));

    // 2. 获取当前能量
    double current_energy = _scheduler->getCurrentEnergy();

    // 3. 触发kernel dispatch，开始新的调度轮次
    MRTKernel *kernel = _scheduler->getKernel();
    if (kernel) {
        kernel->dispatch();
    }
}
```

**关键修复**: getKernel()方法（[gpfp_asap_scheduler.cpp:3800-3803](librtsim/scheduler/gpfp_asap_scheduler.cpp#L3800-L3803)）
```cpp
MRTKernel* GPFPASAPScheduler::getKernel() {
    // 从基类的_kernel dynamic_cast到MRTKernel*
    return dynamic_cast<MRTKernel*>(Scheduler::_kernel);
}
```

### 理论调度时间线（8am测试）

假设收集率 = 0.0974 J/ms，单位能耗 = 0.1J：

```

检查task_0: 能量不足 → 等待
```

**t=2ms**（第1次调度尝试）:
```
能量收集: 2ms × 0.0974 = 0.1948J

getTaskN(0): task_0
  需要: 0.1J <= 0.1948J ✅
  预留: 0.1J，剩余: 0.0948J
  → 调度task_0

getTaskN(1): task_1
  需要: 0.1J > 0.0948J ❌
  → 立即停止，设置能量恢复定时器

等待时间计算:
  需要能量: 0.1J - 0.0948J = 0.0052J
  收集率: 0.0974 J/ms
  等待时间: 0.0052 / 0.0974 = 0.053ms ≈ 5.1ms
  触发时间: 2ms + 5.1ms = 7.1ms ≈ 7ms
```

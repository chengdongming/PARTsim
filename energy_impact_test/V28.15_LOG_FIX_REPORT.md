# ASAP调度器V28.15 - 修复能量预留日志误导性问题

## 问题描述

**用户反馈**: 在V28.15的能量预留日志中，"剩余可用"字段显示的值是错误的/误导性的。

**异常日志示例**:
```
🔒 [V28.13] 预留能量: task_1 预留=0.100000J 总预留=0.100000J 剩余可用=0.092232J
```

**问题**: "剩余可用=0.092232J" 让人误以为能量不足，但实际上：
- 预留**前**的可用能量是 **0.192232J**（足够调度task_1）
- 预留**后**的剩余能量是 **0.092232J**

这个日志只显示了预留**后**的剩余能量，但字段名"剩余可用"容易让人误以为是预留**前**的可用能���。

---

## 根本原因

**代码位置**: [gpfp_asap_scheduler.cpp:2891-2894](librtsim/scheduler/gpfp_asap_scheduler.cpp#L2891-L2894)

```cpp
// 旧代码（V28.13）
SCHEDULER_LOG_INFO("🔒 [V28.13] 预留能量: " + task_name +
                 " 预留=" + std::to_string(unit_energy) + "J" +
                 " 总预留=" + std::to_string(_dispatch_reserved_energy) + "J" +
                 " 剩余可用=" + std::to_string(available_energy - unit_energy) + "J");
```

**问题分析**:
- 日志显示的是 `available_energy - unit_energy`（预留**后**的剩余能量）
- 但字段名是"剩余可用"，容易误解为预留**前**的可用能量
- 实际上预留前的可用能量是 `available_energy`，这才是调度决策的关键

---

## 修复方案（V28.15）

### 代码修改

**修改后的代码**:
```cpp
// 新代码（V28.15）
SCHEDULER_LOG_INFO("🔒 [V28.15] 预留能量: " + task_name +
                 " 预留=" + std::to_string(unit_energy) + "J" +
                 " 总预留=" + std::to_string(_dispatch_reserved_energy) + "J" +
                 " 预留前可用=" + std::to_string(available_energy) + "J" +
                 " 预留后剩余=" + std::to_string(available_energy - unit_energy) + "J");
```

### 改进点

1. **明确区分预留前和预留后**:
   - **预留前可用**: 显示 `available_energy`（调度决策依据）
   - **预留后剩余**: 显示 `available_energy - unit_energy`（预留后剩余）

2. **更清晰的字段名**:
   - 从 "剩余可用" → 改为 "预留前可用" 和 "预留后剩余"
   - 明确表达时间顺序（先预留后剩余）

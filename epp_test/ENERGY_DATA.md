## 能量恢复时间计算

如果初始能量只有 1.0 J（能量不足场景）：

### 场景1: 午夜0点
```
能量缺口: 3.94J - 1.0J = 2.94J
恢复时间: 无法恢复（无太阳能）❌
```

### 场景2: 上午8点
```
能量缺口: 2.94J
恢复时间: 2.94J / 0.097416 J/ms = 30.2 ms
```

### 场景3: 中午12点
```
能量缺口: 2.94J
恢复时间: 2.94J / 0.153090 J/ms = 19.2 ms
```

## 抢占场景能耗分析

### 场景：task_low执行时task_high到达抢占

```
t=0ms:    task_low 开始执行（600ms，需要1.20J）
t=250ms:  task_low 已执行250ms（消耗0.50J）
          task_high 到达，立即抢占 ✅

t=250ms:  task_high 开始执行250ms（消耗0.50J）
t=500ms:  task_high 完成

t=500ms:  task_low 恢复执行剩余350ms（消耗0.70J）
t=850ms:  task_low 完成

总能耗: 0.50J + 0.50J + 0.70J = 1.70J

无抢占时能耗: 1.20J (task_low连续执行)
抢占额外开销: 0.50J (task_high能耗)
```

## 测试建议

### 推荐配置

1. **初始能量**: 5.0 J
   - 可以调度所有4个任务
   - 剩余1.06J用于能量恢复测试

2. **最佳测试时间**: 8:00-12:00
   - 有太阳能收集
   - 可以测试能量恢复机制

3. **避免时间**: 18:00-6:00
   - 无太阳能收集
   - 能量耗尽后无法恢复

### 测试场景

1. **级联调度测试**
   - 初始能量5.0J
   - 验证4个任务能否连续调度

2. **能量恢复测试**
   - 初始能量1.0J
   - 验证能量恢复机制
   - 观察恢复时间

3. **抢占测试**
   - 低优先级任务执行时
   - 高优先级任务到达
   - 验证抢占延迟 < 1ms

4. **长时间运行测试**
   - 仿真时长：5000ms
   - 验证任务周期性到达
   - 验证能量持续收集

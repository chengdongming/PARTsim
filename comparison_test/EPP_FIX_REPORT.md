## 🔧 修复内容

### 修复前的问题

**EPP读取方式**（错误）:
```cpp
int64_t actual_time_ms = time_ms + _start_time_offset;
int64_t minute_of_day = (actual_time_ms % 86400000) / 60000;  // 只取当天分钟数
int line_number = minute_of_day + 2;
```

**结果**:
- 总是读取第720行（第1天12:00）
- 辐照度: 434.50 W/m² ❌
- 能量: 78.21 J ❌

### 修复后的正确方式

**EPP读取方式**（正确）:
```cpp
int64_t actual_time_ms = time_ms + _start_time_offset;
int64_t total_minutes = actual_time_ms / 60000;  // ⭐ 使用总分钟数
int line_number = total_minutes + 2;
```

**结果**:
- 读取第269280行（第187天12:00）
- 辐照度: 939.65 W/m² ✅
- 能量: 169.137 J ✅

## 💡 关键发现

### 1. NASA数据文件结构

```
文件: shenyang_solar_minute.csv
总行数: 532800行 = 370天 × 1440分钟/天

行号对应关系:
- 第2-1441行: 第1天 (00:00 - 23:59)
- 第1442-2881行: 第2天
- ...
- 第268561-270000行: 第187天
  - 第269281行: 第187天12:00 (索引269280)
  - 辐照度: 939.65 W/m²
```

### 2. 时间计算差异

**修复前（EPP错误）**:
```cpp
actual_time_ms = 16113600000  // 第187天12:00
minute_of_day = 16113600000 % 86400000 / 60000
             = 43200000 / 60000
             = 720
line_number = 720 + 2 = 722

// 读取第722行 = 第1天12:00 = 434.50 W/m² ❌
```

**修复后（EPP正确）**:
```cpp
actual_time_ms = 16113600000  // 第187天12:00
total_minutes = 16113600000 / 60000
             = 268560
line_number = 268560 + 2 = 268562

// 读取第268562行 = 第187天12:00 = 939.65 W/m² ✅
```

### 3. 为什么之前以为是第1天？

**EPP的旧逻辑**使用`% 86400000`（一天的毫秒数），这会：
- 忽略"第几天"
- 只保留"当天几点"
- 总是读取前1440行中的数据
- 所以读取的是第1天的数据
